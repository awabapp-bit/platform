# منصة أواب الإلكترونية

منصة مسابقات تعليمية (فيديو + اختبار + نقاط) مبنية بصفحات HTML منفصلة ومرتبطة عبر Firebase.

كل صفحة ملف HTML **مستقل بذاته** (التنسيق والسكربتات المشتركة مضمّنة داخل كل ملف)، فهي تعمل بمجرد فتحها مباشرة من أي مكان، بدون الحاجة لمجلدات `css/` أو `js/` بجانبها. الشيء الوحيد المطلوب بجانب الملفات هو **`logo.png`**.

⚠️ **لا توجد حاليًا أي واجهة لإضافة مسابقات أو دروس أو أسئلة** — تم إزالتها عمدًا من `home.html` و`competition.html` بناءً على طلبك، لحين بناء صفحة مسؤول منفصلة. حاليًا لو فتحت الرئيسية هتلاقيها فاضية لحد ما تُضاف بيانات في `competitions` من صفحة المسؤول (اللي هنعملها بعد كده) أو يدويًا من Firebase Console للاختبار.

## الصفحات

| الملف | الوظيفة |
|---|---|
| `index.html` | تسجيل الدخول |
| `register.html` | إنشاء حساب جديد |
| `home.html` | الصفحة الرئيسية (قائمة المسابقات + النقاط) |
| `account.html` | الحساب الشخصي وتغيير كلمة المرور |
| `competition.html?id=...` | صفحة مسابقة: الدروس، الفيديو، الاختبار |

كل صفحة ملف مستقل، لكنها كلها تقرأ نفس حساب المستخدم من Firebase (نفس تسجيل الدخول ونفس النقاط تظهر في كل مكان).

## قبل التشغيل — خطوتان مهمتان في Firebase Console

### 1) تفعيل تسجيل الدخول بالبريد وكلمة المرور
Firebase Console ← مشروع **awab-34353** ← Authentication ← Sign-in method ← فعّل **Email/Password**.

### 2) رفع قواعد قاعدة البيانات
Firebase Console ← Realtime Database ← Rules ← الصق محتوى ملف `firebase-rules.json` المرفق ← Publish.

هذه القواعد تمنع أي مستخدم عادي من:
- تعديل حقل `isAdmin` الخاص به (لا يمكن لأحد ترقية نفسه لمسؤول من المتصفح).
- الكتابة داخل `competitions` (إضافة/تعديل مسابقات أو دروس أو أسئلة) إلا إذا كان `isAdmin = true`.

بدون هذه القواعد، أي شخص يفتح أدوات المطوّر في المتصفح يقدر يتحايل على فحص المسؤول اللي بيحصل في الواجهة فقط.

## تعيين مستخدم كمسؤول (Admin)

لا توجد صفحة لإنشاء مسؤول من داخل الموقع (لأسباب أمنية). بعد إنشاء الحساب عاديًا من `register.html`:

1. Firebase Console ← Realtime Database ← Data.
2. افتح `users/<uid الخاص بالمستخدم>`.
3. غيّر قيمة `isAdmin` من `false` إلى `true`.

حاليًا لا توجد أزرار إضافة داخل الرئيسية أو صفحة المسابقة؛ ستُستخدم صلاحية `isAdmin` لاحقًا في صفحة المسؤول المستقلة اللي هنبنيها بعد كده.

## شعار المنصة

ضع ملف الشعار باسم **`logo.png`** في نفس مجلد الملفات (بجانب `index.html`). يظهر تلقائيًا في:
- صفحة تسجيل الدخول وإنشاء الحساب.
- الهيدر في كل الصفحات الداخلية.

إذا لم يوجد الملف، يختفي مكان الشعار تلقائيًا دون كسر التصميم.

## هيكل البيانات في قاعدة البيانات

```
users/{uid}
  name, email, points, isAdmin, createdAt

competitions/{compId}
  title, description, coverUrl, createdBy, createdAt
  lessons/{lessonId}
    title, videoUrl
    questions/{questionId}
      text, options[4], correct (0-3), points

userProgress/{uid}/{compId}/{lessonId}
  watched, completed, score, earnedPoints, answeredAt
```

## كيف تعمل رحلة المستخدم

1. **تسجيل الدخول / إنشاء حساب** — يتم حفظ الدخول تلقائيًا على الجهاز (تذكرني)، فلا يحتاج المستخدم لإعادة كتابة بياناته كل مرة.
2. **الرئيسية** — تعرض كل المسابقات التي أضافها المسؤول + نقاط المستخدم في الهيدر.
3. **صفحة المسابقة** — قائمة الدروس، كل درس فيديو يوتيوب. بعد انتهاء الفيديو تلقائيًا يفتح الاختبار.
4. **الاختبار** — عند التسليم تُحسب النقاط وتُضاف لرصيد المستخدم تلقائيًا، ولا تُحتسب مرتين لنفس الدرس.
5. **الحساب الشخصي** — الاسم والبريد فقط، وزر لتغيير كلمة المرور (يتطلب كلمة المرور الحالية + كلمة جديدة مطابقة لنفس الشروط المستخدمة عند التسجيل).

## ملاحظة أمان بسيطة

فحص "هل المستخدم مسؤول؟" لإظهار/إخفاء أزرار الإضافة يتم في الواجهة (JavaScript)، لكن الحماية الحقيقية هي قواعد قاعدة البيانات (`firebase-rules.json`). حتى لو حاول مستخدم عادي استدعاء دوال الإضافة يدويًا من المتصفح، Firebase سيرفض الكتابة لأنه ليس `isAdmin`.

## التشغيل

الملفات كلها HTML/CSS/JS عادية بدون أي أدوات بناء (build tools). يكفي رفعها على أي استضافة ثابتة (Firebase Hosting, Netlify, Vercel, أو حتى فتح `index.html` مباشرة) مع وضع `logo.png` بجانبها.
