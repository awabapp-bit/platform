# منصة أواب الإلكترونية

منصة مسابقات تعليمية (محاضرات فيديو + اختبار + نقاط تراكمية) — موقعان منفصلان يشتركان في نفس قاعدة بيانات Firebase:

- **موقع المستخدم**: مجلد `user/`
- **لوحة تحكم المسؤول**: مجلد `admin/`

كل صفحة HTML مستقلة بذاتها (تنسيقها وسكربتاتها المشتركة مضمّنة داخلها).

---

## الرابط الحي

الموقع منشور على GitHub Pages:

- **موقع المستخدم:** `https://awabapp-bit.github.io/platform/user/`
- **لوحة تحكم المسؤول:** `https://awabapp-bit.github.io/platform/admin/`

بما إن كل مجلد فيه `index.html`، الرابط أعلاه يفتح صفحة الدخول المناسبة تلقائيًا بدون كتابة اسم الملف.

### ⚠️ خطوة إجبارية: إضافة النطاق في Firebase

عشان تسجيل الدخول يشتغل من الرابط ده، لازم تضيف نطاق GitHub Pages كنطاق مصرّح له:

Firebase Console ← مشروع **awab-34353** ← Authentication ← تبويب **Settings** ← **Authorized domains** ← Add domain ← اكتب: `awabapp-bit.github.io`

بدون هذه الخطوة، تسجيل الدخول والتسجيل هيفشلوا برسالة `auth/unauthorized-domain`.

---

## ⚠️ خطوة إجبارية: تحديث قواعد قاعدة البيانات

نسخ محتوى `firebase-rules.json` المرفق ولصقه في:

Firebase Console ← Realtime Database ← تبويب **Rules** ← الصق المحتوى ← **Publish**

هذه النسخة من القواعد (محدّثة) تسمح للمسؤول بـ:
- قراءة كل الحسابات وكل تقدّم المستخدمين (لصفحات الحسابات والمشتركين).
- الكتابة في نقاط أي مستخدم وحالة حظره وتقدّمه (لميزتي إعادة التعيين والحظر الجديدتين).

## تسجيل دخول المسؤول

من `admin/index.html` (أو الرابط المختصر `.../platform/admin/`):

- **اسم المستخدم:** `admin`
- **كلمة المرور:** `123`

**تذكّر الجلسة:** بعد أول دخول، لو رجعت لنفس الرابط تاني (بنفس المتصفح)، هيدخلك مباشرة للوحة التحكم من غير ما يطلب منك تكتب البيانات تاني — لحد ما تعمل "تسجيل خروج" بنفسك، أو تمسح بيانات المتصفح.

خلف الكواليس، البيانات دي بتشغّل حساب Firebase مخصص للمسؤول يُنشأ تلقائيًا أول مرة (مفيش أي إعداد يدوي مطلوب منك في Firebase Console لهذه الخطوة).

**ملحوظة:** جلسة الدخول في المتصفح واحدة بس. لو دخلت لوحة التحكم في نفس المتصفح اللي مسجّل بيه حساب مستخدم عادي، هيتبدّل لحساب المسؤول. استخدم متصفح مختلف أو نافذة تصفح خفي لو عاوز الاتنين في نفس الوقت.

## عرض المحاضرة: الفيديو والاختبار معًا

لما يضغط المستخدم على محاضرة مفتوحة من `user/competition.html`، بينتقل لصفحة `user/lesson.html` المستقلة اللي بتعرض:

1. فيديو اليوتيوب كاملاً في أعلى الصفحة.
2. أسئلة الاختبار مباشرة تحته على نفس الصفحة (مش نافذة منبثقة، ومش لازم ينهي الفيديو الأول).
3. بعد التسليم، تظهر النتيجة والنقاط المكتسبة، وزر للانتقال للمحاضرة التالية.

## الفتح بالترتيب (قفل المحاضرات)

- أي محاضرة معادها (`releaseAt`) لسه ما جاش → تظهر مقفولة بأيقونة ساعة وتاريخ النشر، ولو حاول يدخل رابطها مباشرة هتظهر له نفس رسالة القفل.
- أي محاضرة جه معادها لكن المستخدم لسه ما أنهاش المحاضرة اللي قبلها (فيديو + اختبار) → تظهر مقفولة بأيقونة قفل.
- المحاضرة الأولى دائمًا مفتوحة بمجرد وصول معادها.

هذا الفحص يتكرر في صفحة `lesson.html` نفسها أيضًا (مش بس في قائمة المحاضرات)، فحتى لو حد حاول يفتح رابط محاضرة مقفولة يدويًا، هيشوف رسالة قفل بدل المحتوى.

## إدارة الحسابات (`admin/users.html`)

لكل حساب (غير حساب المسؤول نفسه):

- **إعادة تعيين النقاط**: تصفّر نقاط الحساب بالكامل وتمسح كل تقدّمه في كل المسابقات (يبدأ من جديد). يطلب تأكيدًا قبل التنفيذ ولا يمكن التراجع عنه.
- **حظر / إلغاء حظر الحساب**: الحساب المحظور يُسجَّل خروجه فورًا (لو كان داخل الموقع في تلك اللحظة) ولا يقدر يسجّل دخول تاني إلا بعد إلغاء الحظر، مع رسالة واضحة له بالسبب.

بحث فوري بالاسم أو البريد الإلكتروني، وإحصائية بعدد الحسابات وإجمالي النقاط الموزّعة على مستوى المنصة.

## استجابة الشاشات (لوحة التحكم)

لوحة تحكم المسؤول بالكامل (الشريط العلوي، الجداول، بطاقات المسابقات، صفوف المحاضرات) متجاوبة الآن مع الشاشات الصغيرة (موبايل/تابلت) — الجداول تتمرر أفقيًا عند الحاجة، والأزرار والبطاقات تترتب رأسيًا بدل ما تتكسّر.

## هيكل البيانات في قاعدة البيانات

```
users/{uid}
  name, email, points, isAdmin, blocked, createdAt

competitions/{compId}
  title, description, coverUrl, createdBy, createdAt
  lessons/{lessonId}
    title, videoUrl, releaseAt (timestamp), createdAt
    questions/{questionId}
      text, options[4], correct (0-3), points

userProgress/{uid}/{compId}/{lessonId}
  watched, completed, score, earnedPoints, answeredAt
```

الإضافة الجديدة: `blocked` في سجل المستخدم (true/false).
